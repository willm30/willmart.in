My Projects Stack:

  • Build My Own Website

    I'd wanted to create my own website for a while. Ironically, I had no extrinsic reason to—I just fancied it. To get started, I completed the first three modules on FreeCodeCamp: "Responsive Web Design", "JavaScript Algorithms and Data Structures", and "Front End Development Libraries". This took me about 150 hours (far below the advertised 900). I then read the first 14 chapters of Marijn Haverbeke's "Eloquent Javascript". It progressively appeared to become less and less relevant to my specific objective of building this website, and while a thorough grounding in Javascript fundamentals, beyond my needs. I therefore abandoned it, and any further focused study, in favour of a hands-on approach to learning.


    I eschewed using a CMS such as Wordpress, because I wanted to do everything myself, and retain the ability for total customisation without paying for the privilege. Initially, I struggled to design the site such that I could edit something in one place and update it everywhere else. I had multiple CSS files for multiple HTML files, and when updating one, I’d have to remember to update all the others, and thus mistakes were likely.


    The first minimum viable product took around 10 hours to design and code. It was modelled on a simple blog, with a home page containing links to separate article pages. I kept the styling simple, going for a comic book vibe because it's playful, easy, and quick to mock up. I did, however, splash out on animating the hover transitions and the page’s title. I did the former in CSS, and the latter by writing a simple JavaScript function that loops through an array every 2 seconds and returns a different title by which I aspire to be professionally identified. I also wrote a media query to set a mobile style when accessed from a small screen.


    The second major refactoring occurred when I decided, instead of linking articles in separate HTML files, to animate their appearance in the current window on click. Writing these scripts were time-consuming, messy, and eminently unscalable. I wrote specific animation functions for each individual list item, and then copied the first to create the second and so on, each time making a few adjustments. After achieving the desired effect on desktop, I realised I hadn’t thought about mobile—and thus virtually had to repeat the entire exercise by setting dimension values relatively instead of absolutely. After testing this out on mobile, the animation didn’t perform well on a small screen, so I ditched it altogether—but the effort to make the site responsive wasn’t wasted. At this stage, I also added the clouds in the background


    At this point, I had a responsive, animated site, but the code was still unscalable. If I wanted to add a new list item to the HTML, I would have to go into the JS file and duplicate much of my existing code. Not only this, I would then have to go into the duplicated code and tweak specific identifiers so the new function pointed to the correct list item. This wouldn’t do. To remedy the site’s inflexibility, I refactored the JS I’d written such that I only needed one animator function that takes a project ID as an argument, and with that it expands and collapses the list item to fit the respective article on click. I also made some minor tweaks to the site’s HTML by, for instance, replacing IDs with class names where I was applying the same styles to multiple IDs in CSS. I also added semantic tags such as Main, Section and Header.


    And that takes me up to the site’s current iteration. The code is published on Github if you’d like to see it. The information contained in "Eloquent Javascript" later become more relevant as I began applying it, but I would say the best resources for the nuts-and-bolts building of the site were the blogs I found through Google and StackedOverflow. I’m incredibly fortunate to be learning in a time where I can get highly accurate responses to specific questions instantaneously, and for free. I also use sites such as W3Schools and MDN when searching for method or property references, essentially like I would use a dictionary if I’d forgotten the meaning of a new word.


    The next improvement I would like to make is to include a progress bar that is connected to the spreadsheet in which I track the time spent working on these projects. Currently, I’ve logged 60 hours on the website, and many more on front-end in general. I would like to be able to visualise this data in real time.
