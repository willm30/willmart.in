  • Build My Own Website

    I'd wanted to create my own website for a while. Ironically, I had no extrinsic reason to—I just fancied it. To get started, I completed the first three modules on FreeCodeCamp: "Responsive Web Design", "JavaScript Algorithms and Data Structures", and "Front End Development Libraries". This took me about 150 hours (far below the advertised 900). I then read the first 14 chapters of Marijn Haverbeke's "Eloquent Javascript". It progressively appeared to become less and less relevant to my specific objective of building this website, and while a thorough grounding in Javascript fundamentals, beyond my needs. I therefore abandoned it, and any further focused study, in favour of a hands-on approach to learning.


    I eschewed using a CMS such as Wordpress, something I expect I'll eventually have to rectify, for one reason: I wanted to do everything myself, and have the ability for total customisation without paying for the privilege. As is becoming clear with just three unique web pages, it becomes difficult to manage code and content without a dedicated system in place. Currently, I'm simply duplicating my HTML files for every new page I wish to create, which is problematic, at least because if I change code relating to the structure of the site, I have to manually update every page it affects—and thus mistakes are likely.


    It took about 10 hours to design and code my minimum viable product. My site currently has two styles, one for the Home Page, and one for articles such as this. Every page has a Nav Bar, a Title, and some Content—these constitute the main components of every page. I kept the styling simple, going for a comic book vibe because it's playful, easy, and I'm not yet at the stage where I can design something in XD and translate it into functioning code. I did, however, splash out on the animated title. I wrote a simple JavaScript function that loops through an array every 2 seconds and returns a different title by which I aspire to be professionally identified. I also had to ensure the site was responsive to different screen sizes, and wrote a couple of media queries for smaller screens (which I'm yet to actually test on mobile!). As a finishing touch, I animated the hover selectors on the Home Page links and the Nav Bar so they transition instead of flashing. Much sexier.


    And that's pretty much it. It was a fun project. I learned enough HTML, CSS, and JS to build what you see before you. I think part of my motivation in creating the site was to have a place for publishing content online that felt unique. If this is the metric by which to measure my progress, then my work here is done. I'll continue to iterate in the meantime.


    Post-script: The information contained in "Eloquent Javascript" has since become more relevant as a result of the animations I've created for my main list elements. This added about 15 additional hours onto the project time.


    Post-post-script: After completing the first MVP of the site for desktop, I re-wrote much of the code to make it responsive, so that it would work on mobile. All in all, I have logged 42 hours of work on the website.
